Defines the schema for a type of object (e.g., a node in a graph, a table row).

An ObjectTypeDefinition specifies the structure for a category of data entities. It includes a unique name, an optional description, and a list of `PropertyDefinition`s that define its attributes.

name

A unique name for the object type (e.g., ‘Document’, ‘Person’). Conventionally, PascalCase is used.

Type:

str

description

An optional human-readable description of what this object type represents.

Type:

Optional[str]

properties

A list of properties that define the attributes of this object type.

Type:

List[PropertyDefinition]

Ensures that an ObjectTypeDefinition has at most one primary key.

While the id field from MemoryInstance serves as a unique identifier for ObjectInstance`s, an `ObjectTypeDefinition can optionally define one of its properties as a primary key for domain-specific identification or for database layers that require an explicit primary key column.

Defines the schema for a type of relation between objects (e.g., an edge in a graph).

A RelationTypeDefinition specifies the structure for relationships that can exist between instances of `ObjectTypeDefinition`s. It includes a unique name, allowed source and target object types, and any properties specific to the relation itself (edge properties).

name

A unique name for the relation type (e.g., ‘HAS_AUTHOR’, ‘REFERENCES’). Conventionally, UPPER_SNAKE_CASE is used.

Type:

str

description

An optional human-readable description of what this relation type represents.

Type:

Optional[str]

source_object_type_names

A list of names of `ObjectTypeDefinition`s that are allowed as the source (or “from” side) of this relation.

Type:

List[str]

target_object_type_names

A list of names of `ObjectTypeDefinition`s that are allowed as the target (or “to” side) of this relation.

Type:

List[str]

properties

A list of properties that belong to the relation itself (often called edge properties). Defaults to an empty list if the relation has no properties.
